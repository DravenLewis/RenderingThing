NeoECS

NeoECS is the lightweight ECS framework bundled with RenderingThing in lib/neoecs/include/neoecs.hpp. It provides entities, components, systems, events, and a GameObject wrapper for hierarchical scene graphs.

Create an instance with NeoECS::newInstance(), call init(), then drive updates via update(deltaTime).

Highlights

Managers Included

ECSEntityManager, ECSComponentManager, and ECSSystemManager are created automatically in NeoECS::init().

GameObject Helper

GameObject wraps an ECSEntity and adds parent/child helpers plus component shortcuts.

Event Flow

Neo_EventManager broadcasts ECS lifecycle events like entity and game object changes.

Quick Example

Define a component, register a system, and spawn a game object.

#include "neoecs.hpp"
using namespace NeoECS;

struct Transform : ECSComponent {
    using ECSComponent::ECSComponent;
    float x = 0.0f;
    float y = 0.0f;
};

struct MovementSystem : ECSSystem {
    void updateECSSystem(float dt) override {
        // Pull components from the ECS managers and update them here.
        (void)dt;
    }
};

int main() {
    NeoECS* ecs = NeoECS::newInstance();
    ecs->init();

    auto* api = ecs->getAPI();
    GameObject* player = api->CreateGameObjectAndInstantiate("Player", nullptr);
    player->addComponent();

    ecs->getSystemManager()->addECSSystem(std::make_unique());
    ecs->update(1.0f / 60.0f);

    ecs->dispose();
    return 0;
}

Core Concepts

  • Entities: ECSEntity holds the name, unique id, and parent/child relationships.
  • Components: Derive from ECSComponent to store per-entity state.
  • Systems: Derive from ECSSystem and update via updateECSSystem().
  • Context: ECSContext stores shared data, the daemon ECS instance, and globals.
  • Events: ECS events flow through Neo_EventManager and Neo_ECSEvent types.
